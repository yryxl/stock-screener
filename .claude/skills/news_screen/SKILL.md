# AI 舆情筛股（TODO-004 / REQ-AI-NEWS-001）

## 为什么有这个 skill

**模型是滞后指标，AI 是实时雷达**。

纯数据模型（财务报表、PE 分位、ROE 趋势）的盲区：
- 政策风险：教培双减 2021、互联网反垄断 2021、地产三道红线、医药集采
- 舆情风险：食品安全、药品质量、董事长被查、产品召回
- 行业新闻：AI 算力需求爆发、新能源补贴退坡
- 突发事件：疫情、战争、制裁

模型给出"买入"信号时，AI 必须多做一道**人文筛查**，避免"看着数据漂亮但踩雷"。

## 触发场景

用户在对话里说类似的话：
- "帮我筛今天的候选股"
- "用 AI 筛一下 X / Y / Z 这几只"
- "看看模型推荐的股最近有没有什么新闻"
- "这只股的舆情怎么样"
- "查一下 X 公司最近的政策风险"

输入可以是：
- 几只股票代码（如 600519、000858、603288）
- 模型刚跑完的推荐列表
- 用户从其他渠道看到的股票

## 严格执行原则（必须遵守）

### 1. 宁可错过不犯错
即使模型给"强烈买入"信号，只要发现重大利空（造假/丑闻/政策剧变），结论必须是 🚨 跳过，不要被模型信号带偏。

### 2. 不替用户决策
舆情判断主观性强，**给证据 + 分类，让用户自己定**。不要用"建议买入"等强决策语言，用"无风险信号"/"建议复查"/"强烈建议跳过"等中性表述。

### 3. 必须引用源 + AI 幻觉防范（5 条硬性要求）
- **多源原则**：每个关键结论必须 ≥ **3 个独立源**，否则标"不确定"
- **权威源优先级**：证监会公告 > 上市公司年报/招股书 > 主流财经媒体（财新/财联社/经济观察报）> 普通自媒体
- **时间戳必显**：每条引用必须带日期，**2 年前的数据明确标"老数据"**
- **数字交叉**：声称"营收 X 亿"必须从 2 个独立渠道验证（年报+招股书 或 年报+招标文件）
- **AI 自检**：每次输出强制问自己"这条结论我是不是在编？引用源能否被用户点击验证？"
- 没有引用源的判断 = 不可信，**禁止给出**

### 4. 时间窗口控制
- 公司负面新闻：最近 **30 天**
- 行业政策：最近 **90 天**
- 财报预告/突发公告：最近 **15 天**
- 上下游交叉验证：上下游公司**最近一年财报**
搜索时一定带"2026 年"或"近 30 天"等时间限定，避免拉到老新闻。

### 5. A 股特殊关注点
按 REQ-152 政策风险标签分级关注：
- 高风险行业（地产/教培/医药/互联网平台/游戏）：政策搜索权重 ×2
- 中风险（消费/科技/新能源/军工）：政策 + 行业新闻并重
- 低风险（白酒/银行/公用事业/食品）：主要看公司层面

### 6. A 股造假率高，必查 5 大警告信号（康得新/康美/瑞幸教训）
即使报表数据漂亮，仍要交叉核查：
1. 高现金 + 微薄利息收入（康得新 150 亿现金但利息几乎 0 → 钱可能假）
2. 高毛利 + 低应收（货真的运了吗）
3. 经营现金流 < 净利润（长期）
4. 应收账款增速 > 营收增速 1.5x
5. 关联交易掏空

## 标准流程（每只候选股）

### 步骤 1：拉模型基础信息
如果用户提供了股票代码但没说细节，先看 daily_results.json / holdings.json / watchlist.json 是否有该股的模型评估（信号、PE、ROE、行业等）。
有的话以模型结果为基础，AI 只补充舆情。
没有的话提示用户"该股不在模型扫描范围，AI 仅做舆情分析"。

### 步骤 2：3 层搜索（用 WebSearch）

**A. 公司层**（最重要）：
```
搜索词模板：
  "{公司名} 最近 30 天 负面新闻"
  "{公司名} 2026 财报"
  "{公司名} 董事长 OR 高管 减持 OR 被查"
  "{公司名} 产品召回 OR 质量问题 OR 立案"
```

**B. 行业层**：
```
搜索词模板：
  "{行业} 政策 2026"
  "{行业} 监管 利空"
  "{行业} 集采 OR 反垄断 OR 限制"
```

**C. 时点层**：
```
搜索词模板：
  "{公司名} 业绩预告 2026"
  "{公司名} 突发 公告"
  "{股票代码} 复牌 OR 停牌 OR 异动"
```

**D. 上下游数据交叉验证（浑水式，TODO-037）**：
学浑水做空东方纸业的方法——查上游原料供应商交易记录而不是下游销售。
```
搜索词模板：
  # 上游：公司声称的"主要供应商"是真的吗
  "{公司名} 主要供应商 2026 年报"
  "{供应商名} 客户 {公司名}"
  "{行业} 上游原料 总规模 2026"  # 行业总产能能否支撑

  # 下游：公司声称的"主要客户"能否被验证
  "{公司名} 主要客户 2026 年报"
  "{客户公司} 采购 {公司名}"  # 客户的财报里有没有这家供应商

  # 关联交易：供应商 vs 公司是否有关联
  "{供应商公司} 工商信息 注册地址 注册资本"
  "{公司名} 关联交易"
```

**判定要点**：
- 公司声称卖了 100 亿给客户 A，但客户 A 财报里只采购 10 亿 → 🚨 严重矛盾
- 公司声称从供应商 B 采购 5000 万，但供应商 B 注册资本只 100 万 → 🚨 不可能
- 行业上游总产能 200 亿，公司声称采购 50 亿（占 25%）但市占率仅 5% → 🚨 不合理

**E. 实地调研（巴菲特 scuttlebutt，TODO-037）**：
散户去不了工厂，但 AI 可以查网友/媒体的实地考察。
```
搜索词模板：
  "{公司名} 工厂 实地考察 视频"
  "{公司名} 门店 客流 探访"
  "{公司名} 招聘 网传 离职 员工"  # 内部声音
  "{产品名} 真实评价 用户"  # 终端反馈
```

**判定要点**：
- 实地视频/小票/招聘信息 vs 财报数字一致 → ✅
- 实地反映冷清，但财报营收高速增长 → ⚠ 或 🚨（瑞幸式）
- 员工大规模离职/吐槽 + 财报漂亮 → ⚠

**F. 管理层"言行核查"（TODO-038，巴菲特管理层 5 标准）**：
当前模型只看质押/商誉/分红 3 维。AI 补 2 维：
```
搜索词模板：
  "{公司名} 董事长 历年 业绩承诺"
  "{公司名} 5 年 分红 OR 回购 历史"
  "{公司名} 收购 OR 并购 历史"  # 是否盲目扩张
  "{公司名} 实控人 减持 套现 记录"
```

**判定要点**：
- 历年承诺业绩 vs 实际履约（连续不达预期 → 警惕）
- 5 年是否有股票回购注销（巴菲特：分红+回购才是真为股东）
- 频繁高溢价收购（盲目跟风症状）
- 大额减持时点（出货 vs 真心看好）

每只股建议至少 6 次 WebSearch，覆盖 A-F 六层（公司 / 行业 / 时点 / 数据交叉 / 实地 / 管理层）。

### 步骤 3：（可选）深度阅读关键文章
若 WebSearch 摘要里出现高度相关的标题，用 WebFetch 拉全文确认细节。
重点关注：
- 监管/证监会/银保监会发文
- 财报预告中的关键数字
- 公司被立案/限制减持等具体内容

### 步骤 4：输出分类 + 证据

**分类标准**（保守优先）：
- 🚨 **重大利空**（强烈建议跳过，即使模型说买）：
  - 财务造假被立案
  - 董事长/实控人被查/失联
  - 行业整体被新政策冲击（教培、地产式）
  - 重大产品安全事故
- ⚠ **有争议**（用户自己判断）：
  - 单条负面新闻但未实锤
  - 行业短期利空（不至于颠覆）
  - 高管减持但未集中
  - 财报预告偏差但仍正
- ✅ **无风险信号**（可以按模型信号执行）：
  - 30 天无负面新闻或仅常规波动
  - 行业政策稳定
  - 财报符合预期

## 局限说明（必须告诉用户）

每次输出时附上以下声明：

> **⚠ 本筛查的局限**：
> - AI 不是先知，新闻可能滞后或片面
> - WebSearch 结果受网络/源稳定性影响
> - 舆情判断主观性强，同一条新闻不同人解读不同
> - **最终决策权在你**，AI 提供的是辅助证据，不是结论

## 错误处理

- WebSearch 拉不到结果 → 明确告知"该股 30 天无公开负面新闻，但不代表无风险"
- 网络/接口失败 → 跳过该股的网络层，仅给模型信号
- 用户没给代码只给名称 → 先用 WebSearch 查代码再继续

## 不要做的事

- ❌ 不要在没有 WebSearch 引用的情况下给"分类"结论
- ❌ 不要替用户做"买/卖"决策（用"建议执行/跳过"等中性词）
- ❌ 不要拿过时新闻当最新证据（注意时间戳）
- ❌ 不要为了"有内容"而堆砌不相关的新闻
- ❌ 不要忽视 🚨 信号（即使模型强烈推荐也要跳过）

## 与现有模型的协同关系

| 维度 | 模型负责 | AI 舆情筛查负责 |
|---|---|---|
| 财务健康度 | ROE / 负债率 / 现金流 | - |
| 估值 | PE / CAPE / 行业区间 | - |
| 长期质量 | 十年王者 / 印钞机 / 护城河 | - |
| **实时风险** | - | **政策/舆情/突发事件** |
| **模型熔断** | REQ-151（健康度自检）| - |
| **最终判断** | 模型给信号 → AI 加舆情 → 用户决定 |

- 每次跑完 skill 后，AI 可以问用户："发现的关键新闻要不要记入 black_swan_events.json"
- 反复出现的舆情风险（如某行业被持续监管）应建议加入 REQ-152 政策风险标签
