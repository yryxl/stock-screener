📖 本页面的使用方法

什么时候查？

建议每月查一次，或者在市场发生重大变化（黑天鹅事件、连续大涨大跌）后查。

红灯什么意思？

如果任意指标显示 🔴 警示，表示模型可能已经失效。这时：

  • 暂停 按模型信号操作
  • 回归 定投宽基 ETF（沪深 300 + 纳指）
  • 等所有指标回到绿灯再继续使用模型

各指标的含义

  • 持仓胜率：当前持仓中盈利的股票占比。长期低于 40% 说明选股有问题。
  • 最大回撤：当前最惨一只股票的亏损幅度。超过 -30% 要警惕。
  • 3 月买入准确率：模型推荐买入的股票 3 个月后涨幅为正的比例。低于 55% 说明模型预测失效。需历史快照≥ 3 个月才能算。
  • 跑赢沪深 300：模型推荐组合对比沪深 300 的超额收益。短期参考，连续 3 年跑输就应该用指数替代。
  • 信号矛盾数：逻辑一致性测试检测出的文案矛盾数量。应为 0。
  • 已知 Bug 数：TESTING.md 中记录的 bug 数量。
  • 🦢 黑天鹅状态（REQ-151 规则 B）：当前日期是否落在已记录的黑天鹅事件窗口（疫情/政策剧变等）。触发时建议主动降仓。事件配置在 black_swan_events.json。
  • 📉 长期亏损股（REQ-151 规则 C）：持仓中"持有 >3 年且累计收益为负"的股票。需要 holdings.json 里有 buy_date 字段。触发时强制复查"是真错还是假错"。
  • 📊 连续跑输沪深 300（REQ-151 规则 A）：滚动 3 年模型组合 vs 沪深 300。判定标准是时间跨度（最早快照≥ 3 年）+ 密度检查（防止数据残缺误判）。触发时建议暂停模型转定投宽基。

3 条熔断规则的设计依据

REQ-151 设计原则：模型基于巴菲特/芒格美股理念 + A 股回测推出，可能"水土不服"。3 条规则各管一种失效模式：

  • 规则 A 处理"模型规则在 A 股某段时间不适用" → 用结果（是否跑赢沪深 300）说话
  • 规则 B 处理"突发黑天鹅" → 时间窗口检测，自动告警
  • 规则 C 处理"个股长期蛀虫" → 强制复查，避免侥幸持有

详细背景见 docs/REQUIREMENTS.md REQ-151（按 7 要素模板撰写）

局限

本监控只是"统计层面"的健康度。以下风险它不能检测：

  • 政策冲击突然发生（见黑天鹅事件列表）
  • 整个市场进入 20 年长熊（日本式衰退）
  • AI 革命颠覆某行业 ROE 基础

这些"范式转换"类风险，需要你自己的判断。模型不是神。
